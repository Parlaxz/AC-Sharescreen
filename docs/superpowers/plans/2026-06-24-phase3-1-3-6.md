# Phase 3.1–3.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Phase 3 group runtime, synchronization, active-stream ownership, and host↔viewer watch flow execute end-to-end in production code instead of stubs.

**Architecture:** Keep the existing Phase3Runtime-centered design, but move from placeholder wiring to authenticated, persisted, per-viewer runtime ownership. Runtime initialization becomes serialized and StrictMode-safe; group sync becomes schema-validated and durable; stream/media flows become exact-peer, exact-sender, exact-viewer operations instead of broad broadcasts and fallback lookups.

**Tech Stack:** Electron, React 19, Zustand, Vitest, Zod, VDO.Ninja SDK wrapper, shared CRDT/group protocol modules.
